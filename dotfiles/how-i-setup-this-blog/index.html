    How I Setup This Blog¶

    I recently switched my pkm system from a Joplin / OneNote hybrid to one centered around Obsidian. My initial foray into the world of roam-like pkm tools started with logseq, however, what ultimately led me to Obsidian was the maturity of its vim binding implementation. No amount of configuration made logseq's vim bindings comfortable enough for the tool to feel non-intrusive and effortless.

    Soon after, I stumbled across an HN post by mr-karan which mentioned leveraging mkdocs-material for static site generation, and here we are.

    10000-ft View¶

    • Obsidian used for writing notes.
    • GitHub used for Obsidian vault storage.
    • MkDocs-Material used for .md to .html conversion.
    • GitHub Pages used for hosting.

    main.yml¶

    Any time I commit and push from Obsidian to my notes repository, my site is automatically rebuilt and deployed. This is orchestrated using GitHub Action workflows in both the notes repository and the mkdocs-obsidian repository.

    main.yml contains the code for the GitHub Action workflow which triggers the "Build and Deploy Site" workflow in the mkdocs-obsidian repository.

    name: Trigger Deployment 
on:
  push:
    branches:
      - main

env:
  USER: peddamat
  REPO: mkdocs-obsidian

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      # Checks-out your repository under $GITHUB_WORKSPACE, so your job can access it
      - name: Trigger Build and Deploy
        run: |
          curl -X POST https://api.github.com/repos/$USER/$REPO/dispatches \
          -H 'Accept: application/vnd.github.everest-preview+json' \
          -u ${{ secrets.API_TOKEN_GITHUB }} \
          --data '{"event_type": "Trigger Workflow", "client_payload": { "repository": "'"$GITHUB_REPOSITORY"'" }}'
      - uses: actions/checkout@v3

    main.yml¶

    name: Build and Deploy Site

on:
  repository_dispatch:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout mkdocs-obsidian repo
        uses: actions/checkout@v3

      - name: Checkout notes repo into ./notes
        uses: actions/checkout@v3
        with:
          token:  ${{ secrets.PULL_NOTES }}
          repository: peddamat/notes        
          path: notes

      - name: Setup Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.8'

      - name: Upgrade pip
        run: |
          # install pip=>20.1 to use "pip cache dir"
          python3 -m pip install --upgrade pip

      - name: Get pip cache dir
        id: pip-cache
        run: echo "::set-output name=dir::$(pip cache dir)"

      - name: Cache dependencies
        uses: actions/cache@v2
        with:
          path: ${{ steps.pip-cache.outputs.dir }}
          key: ${{ runner.os }}-pip-${{ hashFiles('**/requirements.txt') }}
          restore-keys: |
            ${{ runner.os }}-pip-

      - name: Install dependencies
        run: python3 -m pip install -r ./requirements.txt

      - run: mkdocs build

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          personal_token: ${{ secrets.PULL_NOTES }}
          external_repository: peddamat/peddamat.github.io
          publish_branch: main  # default: gh-pages
          publish_dir: ./site
          cname: samrambles.com

    mkdocs-flatten-filenames¶

    import os
import sys
from timeit import default_timer as timer
from datetime import datetime, timedelta

from mkdocs import utils as mkdocs_utils
from mkdocs.config import config_options, Config
from mkdocs.plugins import BasePlugin
from mkdocs.structure.files import Files

import logging
import mkdocs.utils
log = logging.getLogger(f"mkdocs.plugins.{__name__}")
log.addFilter(mkdocs.utils.warning_filter)

class YourPlugin(BasePlugin):

    config_scheme = (
        ('param', config_options.Type(str, default='')),
    )

    def __init__(self):
        self.enabled = True
        self.total_time = 0

    def on_files(self, files, config):
        for f in files:
            if f.is_documentation_page():
                f.abs_dest_path = f.abs_dest_path.replace(" ", "-").lower()
                f.dest_path = f.dest_path.replace(" ", "-").lower()
                f.url = f.dest_path.replace("%20", "-").lower()
            log.warning(f"{f.abs_dest_path}")

        return files

    Forking MkDocs¶

    How to specify a GitHub repo in requirements.txt¶

    • Specify commit hash (41b95ec in the context of updated requirements.txt): 

      package-one==1.9.4
git+https://github.com/path/to/package-two@41b95ec#egg=package-two
package-three==1.0.1

    • Specify branch name (master): 

      git+https://github.com/path/to/package-two@master#egg=package-two

    • Specify tag (0.1): 

      git+https://github.com/path/to/package-two@0.1#egg=package-two

    • Specify release (3.7.1): 

      git+https://github.com/path/to/package-two@releases/tag/v3.7.1#egg=package-two

    Note that #egg=package-two is not a comment here, it is to explicitly state the package name - source

    Setting up GitHub Pages¶

    GitHub pages allows setting up a domain based on your GitHub account's username.

    In my case, this means my website is hosted at: peddamat.github.io, and my repository is at github.com/peddamat/peddamat.github.io.

    I used the template provided by jobindjohn/obsidian-publish-mkdocs as a starting point.

    The template consists of two parts: - A MkDocs configuration file: mkdocs.yaml - A GitHub Actions workflow: ci.yml
