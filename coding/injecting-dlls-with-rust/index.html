    DLL Injection¶

    In Part 1 we used Process Hacker to inject hello.dll into the notepad.exe process. The dll-syringe crate allows us to do this programatically with Rust.

    Let's create a hello_inject crate to do this:

    cargo new hello_inject

    First, add the following dependencies to Cargo.toml:

    [dependencies]
hello = { path = "../hello" }
dll-syringe = "0.15.0"

    Let's do a cargo check:

    $ cargo check
   Compiling proc-macro2 v1.0.51

    rustup install nightly

    rustup default nightly

    # Set stable as default
rustup default stable

    We can keep updated by:

    rustup update

    Run the code using:

    cargo build

    Cool.

    Now update main.rs with the following (taken straight from README.MD):

    use dll_syringe::{Syringe, process::OwnedProcess};
use std::{thread, time};

fn main() {
    // find target process by name
    let target_process =
        OwnedProcess::find_first_by_name("notepad.exe").unwrap();

    // create a new syringe for the target process
    let syringe = Syringe::for_process(target_process);

    // inject the payload into the target process
    let injected_payload = syringe.inject("hello.dll").unwrap();
    println!("DLL injected successfully!");

    // do something else
    let ten_millis = time::Duration::from_secs(30);

    println!("Sleeping for 30 secs...");
    thread::sleep(ten_millis);

    // eject the payload from the target (optional)
    syringe.eject(injected_payload).unwrap();
}

    Getting Production Ready¶

    When injecting DLLs, you need both 32-bit and 64-bit versions of the DLL, which requires installing both the 32-bit and 64-bit rust toolchains.

    # Install 32-bit toolchain & target
rustup target add i686-pc-windows-msvc

# Build 32-bit binaries
cargo build --target=i686-pc-windows-msvc

# Build 64-bit binaries
cargo build --target=x86_64-pc-windows-msvc
